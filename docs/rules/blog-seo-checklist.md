# Blog SEO Checklist

## Search intent

Before writing, identify the search intent:

- Informational
- Cost research
- Local service research
- Comparison
- Problem solving
- Pre-estimate research
- DIY research
- Repair decision research

The article must satisfy the intent before pushing a quote request.

## Main keyword

Use the main keyword naturally in:

- SEO title
- H1
- Intro
- At least one H2 if natural
- Meta description
- URL slug when creating a new page

Do not stuff the keyword into every heading.

## Related terms

Use related terms naturally when relevant:

For popcorn ceiling content:
- popcorn ceiling removal
- stucco ceiling removal
- textured ceiling removal
- smooth ceiling
- ceiling refinishing
- skim coating
- sanding
- primer
- flat ceiling paint
- painted popcorn ceiling
- unpainted popcorn ceiling
- ceiling repair

For drywall content:
- drywall repair
- drywall patch
- drywall installation
- drywall tape
- joint compound
- sanding
- primer
- wall repair
- ceiling repair

For painting content:
- interior painting
- wall painting
- ceiling painting
- primer
- prep work
- cut lines
- finish coat

## SEO title

The SEO title should:

- Include the main topic naturally
- Include the city only when local
- Be clear
- Not be clickbait
- Usually stay under about 60 characters when possible

Examples:

- Popcorn Ceiling Removal Cost in Mississauga
- Painted Popcorn Ceiling: Remove or Skim Coat?
- Why Drywall Patches Show After Painting
- How Long Does Popcorn Ceiling Removal Take?

## Meta description

The meta description should:

- Be roughly 140-160 characters when possible
- Explain what the reader will learn
- Include service/city naturally
- Avoid fake guarantees
- Avoid keyword stuffing

## H1

Use one H1 only.

The H1 should clearly match the article topic.

Examples:

- Popcorn Ceiling Removal Cost in Mississauga
- Should You Remove or Skim Coat a Painted Popcorn Ceiling?
- Why Drywall Patches Show Through Paint

## Headings

Use:

- H2 for major sections
- H3 for subtopics
- Logical heading order
- Clear homeowner-friendly wording

Avoid:

- Repeating the same keyword in every heading
- Skipping heading levels
- Using headings only for SEO instead of readability

## Intro

The intro must:

- Answer the reader's concern quickly
- Mention the topic naturally
- Explain what the article covers
- Avoid long generic openings

## Short answer block

Near the top, include a short answer section.

It should be 2-5 sentences.

It should give the direct answer before the detailed explanation.

## Content originality

Do not copy:

- Competitor text
- Competitor headings too closely
- Competitor examples
- Manufacturer text
- Government text
- Other EPF pages word-for-word

Each article must be original and useful.

## Helpful content standard

The article must be written for people first.

It should show:

- Real experience
- Practical knowledge
- Clear explanation
- Honest limitations
- Useful advice
- Specific details
- Clear next step

Avoid:

- Thin AI content
- Repeating obvious statements
- Unnecessary fluff
- Over-optimized keyword repetition
- Fake expertise

## Authority-building SEO

Use SEO practices that improve usefulness, topical authority, and internal link flow.

Every full blog should:

- Be at least 2,200 words unless the user explicitly requests a short support article.
- Cover the topic more completely than generic competitor pages.
- Include a direct short answer near the top.
- Include practical trade details that prove real service knowledge.
- Include related entities and terms naturally, not as keyword stuffing.
- Include original examples based on real renovation logic, not fake case studies.
- Include an FAQ section that answers long-tail homeowner questions.
- Include a featured image with descriptive alt text.
- Include Article or BlogPosting schema if supported.
- Include FAQPage schema only when visible FAQs are present and the project supports it.
- Link to the correct pillar/money page.
- Link to related cluster posts.
- Link to adjacent services only when the next step is useful for the reader.

Authority signals to include when relevant:

- Finish level explanation
- Materials and methods
- Prep and protection
- Dust-control process
- Common mistakes
- Pricing factors
- Timeline factors
- DIY limitations
- Safety cautions
- Photo/quote checklist
- Local service context without fake local claims

## Local SEO

For city blogs:

- Use city name naturally.
- Link to the city service page.
- Link to the main service page.
- Include local context only when accurate.
- Do not invent neighbourhood facts.
- Do not create the same article with only city names changed.
- Change examples, FAQs, CTA, and content angle for each city.

## Internal links

Every blog should include:

- Main service page link when relevant
- Relevant city/service page link when local
- Related service page links where useful
- Related blog links where useful
- Contact/quote CTA link where useful

Use descriptive anchor text.

Do not use "click here."

## External links

Use external links only when useful and trustworthy.

Good external links include:

- Government safety information
- Manufacturer technical information
- Google Search guidance for SEO-related articles
- Building/safety references when relevant

Do not link to competitors in published blog content unless specifically requested.

## FAQ

Add FAQ when useful.

FAQ should:

- Answer real homeowner questions
- Be specific
- Avoid repeating the article word-for-word
- Support long-tail search intent
- Include service/city naturally only when relevant

## Schema

If the project supports structured data, add Article or BlogPosting schema.

For FAQ sections, add FAQPage schema only if the project supports it and the FAQ content is visible on the page.

Article schema should include when possible:

- headline
- description
- datePublished
- dateModified
- author or organization
- image
- mainEntityOfPage

## URL slug

Use lowercase hyphenated slugs.

Examples:

- /blog/popcorn-ceiling-removal-cost-mississauga/
- /blog/painted-popcorn-ceiling-remove-or-skim-coat/
- /blog/why-drywall-patches-show-through-paint/

Avoid:

- Long messy slugs
- Dates in slugs unless needed
- Random numbers
- Keyword stuffing

## Final SEO check

Before finishing, confirm:

- Article is at least 2,200 words unless a shorter format was explicitly requested
- SEO title created
- Meta description created
- One H1 only
- H2/H3 structure is clear
- Main keyword is natural
- Internal links added
- Pillar/money page linked
- Topical cluster links added
- CTA added
- FAQ added if useful
- Featured image or image prompt added
- Schema added if supported
- Image alt text added if images exist
- No duplicate/city-swapped content
- No fake claims
- No copied competitor wording
