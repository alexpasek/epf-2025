# Blog Review Checklist

Before finishing any blog task, complete this checklist.

## Content check

Confirm:

- The article is at least 2,200 words unless the user explicitly requested a short article.
- The article answers the main question early.
- The article is useful to a real homeowner.
- The article matches EPF Pro Services' actual services.
- The article uses practical contractor knowledge.
- The article avoids generic AI filler.
- The article avoids copied competitor wording.
- The article avoids unsupported claims.
- The article avoids fake project examples.
- The article avoids fake statistics.
- The article avoids exaggerated marketing language.
- The article fits a clear topical cluster.
- The article does not expose internal SEO planning notes to readers.
- The article is human-readable and useful without sounding like an SEO report.
- The article adds useful depth instead of filler to reach length.

## SEO check

Confirm:

- Article is at least 2,200 words unless a shorter format was explicitly requested.
- SEO title is included.
- Meta description is included.
- One H1 only.
- Direct answer appears within the first 150 words.
- H2/H3 structure is logical.
- Main keyword is used naturally.
- Related terms are used naturally.
- Intro is clear.
- Short answer section is near the top.
- FAQ is added when useful.
- CTA is added.
- Featured image or image prompt is included.
- URL slug is clean if a new page was created.
- Article/BlogPosting schema is added if the project supports it.
- BreadcrumbList schema is added when the layout has breadcrumbs.
- Service schema is added for service-support blogs when appropriate and supported.
- FAQPage schema is added only if supported and visible FAQ exists.
- FAQ schema matches the visible FAQ exactly.
- Main entity, service type, city, and related terms are used naturally.
- The article covers pricing, process, mistakes, DIY limits, professional help, and quote-prep details where relevant.

## Local SEO check

For local blogs, confirm:

- City is used naturally.
- Matching city/service page is linked.
- Main service page is linked.
- No fake neighbourhood claims.
- No fake local statistics.
- No city-swapped duplicate content.
- Examples and FAQs are adjusted to fit the specific local topic.

## Internal linking check

Confirm:

- Topical cluster is identified.
- Pillar page or money page is linked.
- Pillar/money page is linked in the first third where natural.
- No visible article heading says "Topical cluster," "Search intent," "Target keyword," "Pillar page," "Money page," "SEO notes," "Internal linking plan," or "Competitor research."
- Main service page link added.
- City service page link added if local.
- Related blog links added where useful.
- 2-4 same-cluster blog links are added where available.
- Related links support the same cluster before broader adjacent topics.
- Contact/quote page link added near CTA if available.
- Anchor text is descriptive.
- No "click here" links.
- No irrelevant internal links.
- No excessive linking.

## Image check

For new blogs, confirm:

- Image plan created.
- Featured image included, generated, or prompt created.
- Supporting images included or prompts created.
- File names are lowercase and hyphenated.
- Alt text describes the actual image.
- Every image has unique alt text.
- Captions are added where helpful.
- No fake logos.
- No text baked into images unless requested.
- No unsafe work shown.
- Images match the blog topic.

## Technical check

Confirm:

- Correct route/path.
- Correct imports.
- Existing blog layout reused when possible.
- No duplicate components created.
- No unrelated files changed.
- No obvious build errors.
- Mobile layout considered.
- Metadata pattern matches existing project.
- Images use existing image component/pattern where available.
- New blog visually matches the field-guide design benchmark from `/blog/why-drywall-patches-show-through-paint/`, adapted to the current service topic.
- Hero includes breadcrumb, H1, direct answer, primary CTA, phone CTA, trust badges, and image.
- Quick answer card appears directly after the hero.
- Diagnostic table/checklist/comparison appears near the top.
- Field note, local service block, author/trust block, related article cards, and bottom CTA are present where relevant.
- Hero CTA, mid-page CTA, and bottom CTA are present where layout supports it.
- Related articles are displayed as cards where layout supports it.
- Tables stack cleanly on mobile.

## Final response required

At the end of every blog task, report:

- Files changed
- Page path
- Target keyword
- Blog type
- Search intent
- Internal links added
- Images or image prompts added
- SEO checks completed
- Technical/build checks performed
- Manual review needed
